[https://qa-tool-box.lovable.app/](https://qa-tool-box.lovable.app/)

## What QA Should Inspect

- Test data generator behavior and defaults
- Event, ticket, customer, code, and discount output formats
- Copy/export workflows and generated text quality
- Local storage behavior for saved or locked fields
- Date handling, normalization, and edge cases
- UI workflows across generator pages and helpers
- Build, lint, and local development scripts

## QA Notes

- Reference this repo when QA work needs repeatable test data or helper utilities.
- Compare generated values with backend validation before using them in test cases.
- Keep helper-tool notes focused on behavior, risks, and repeatable checks.
