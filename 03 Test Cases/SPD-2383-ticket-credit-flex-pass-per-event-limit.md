## Qase Draft

Created as [SPT-4978](https://app.qase.io/case/SPT-4978) in Core Checkout suite 597.

TC-1: Core - Checkout - Verify Flex Pass credits scale with purchased quantity per event

**Description:** Verify that an authenticated customer who purchases multiple Flex Passes in one new order receives the correct total Ticket Credit balance and can redeem one credit per purchased pass on the same eligible event. This protects against a combined Ticket Credit being restricted to one redemption for the entire order instead of one redemption per purchased pass.

Additional new-purchase edge checks:

- Split the scaled per-event allowance across multiple completed orders for the same eligible event; allow uses up to the cumulative limit and block the next use without reducing the balance.
- Redeem the credit across multiple eligible events until its total balance is exhausted; reduce the balance accurately after each completed order.
- Complete separate Flex Pass purchases; give each newly generated Ticket Credit the correct total balance and per-event allowance.
- Configure a Limit Per Event, Per User greater than one; scale the generated per-event allowance by the purchased quantity without exceeding the total balance.
- Leave Limit Per Event, Per User empty; do not introduce an event cap, while still enforcing the total balance.
- Try the credit on an ineligible event or ticket type; reject it without reducing the balance.

**Preconditions:**

- An organizer has a Flex Pass purchase event and at least one separate redemption event.
- The Flex Pass ticket type issues three Ticket Credit uses per purchased pass.
- The Ticket Credit has Limit Per Event, Per User set to one and allows the redemption event's target ticket type.
- A fresh authenticated customer has no existing copy of this Ticket Credit.

**Postconditions:** The completed redemption order contains one discounted ticket per purchased Flex Pass, and the customer's remaining Ticket Credit balance reflects one use per pass on that event.

**Tags:** checkout, credits, edge-case

**Parameters:**
PassQuantity: Two, Four

**Steps:**

| Step Action | Data | Expected Result |
| --- | --- | --- |
| Sign in as the prepared customer and purchase the selected quantity of the Flex Pass ticket type in one order. | PassQuantity parameter | The order completes and Ticket Credit uses are issued to the customer for every purchased Flex Pass. |
| Open the customer's Ticket Credits and locate the credit generated by the new Flex Pass order. |  | The available total equals three uses for each purchased Flex Pass. |
| Open the eligible redemption event and add one ticket for each purchased Flex Pass. |  | The basket contains the selected number of eligible tickets. |
| Apply the generated Ticket Credit during checkout. |  | One credit applies to every ticket in the basket, up to the selected PassQuantity. |
| Complete the redemption purchase. |  | The order completes with every selected ticket discounted by the Ticket Credit. |
| Review the completed order and the customer's remaining Ticket Credit balance. |  | The order shows the correct number of applied credits and the remaining balance equals two uses for each purchased Flex Pass. |
