The active `/goal` objective should not say to stop when source-code writes would be required. That wording creates false blocked states. If a loop proposes "write the Playwright test", "change a fixture", "edit app code", or "write Qase", convert that into a handoff note. The handoff should name the future file, assertion, fixture, and review criteria, then the worker should continue to another read-only planning target.

Next useful work requires one of:
- refining the next highest-value planning packet
- discovering a source-backed checkout entry path that could change the ranking
- filtering existing local/read-only Qase output for coverage evidence
- tightening fixture, data, assertion, or review criteria for a planning-ready packet
- product or business-priority clarification only when source evidence cannot answer the question

Do not ask to implement ready packets from this worker. A ready packet is queued for later human or separately authorized implementation while the loop continues planning.

Do not park a candidate only because it needs Playwright implementation, source changes, fixture changes, or Qase writes. Park only when the planning branch needs product or business confirmation, provider/hardware strategy, secrets, destructive setup, or missing read-only access/context.
